# 工作 Skill：赛博哈苏纯前端搭建流程

这个文件用来把本次工作沉淀成可复用路径，后面每次迭代都可以直接沿着它走，避免重复折腾。

## 适用场景

- 想做一个极简、零后端、可直接部署到 GitHub Pages 的图片工具
- 想强调隐私，不上传用户文件
- 想用很少的文件，做出完成度很高的视觉产品

## 本次有效路径

1. 先锁定产品内核，只保留上传、渲染、导出三个动作。
2. 视觉先于功能，先把黑金气质、留白比例、底部信息结构固定下来。
3. 为了控制体积，不盲目引第三方库；能自己写的小逻辑就自己写。
4. 先保证“无 EXIF 也能出片”，再去做“有 EXIF 更高级”。
5. SEO/GEO 不额外做内容页，而是埋进 `meta`、README、manifest 和页面底部说明区。

## 这次沉淀出的实现模板

- 页面骨架：`index.html`
- 视觉语言：`styles.css`
- 本地图片处理与海报生成：`app.js`
- PWA 描述与搜索词承接：`manifest.webmanifest`
- 对外解释与搜索承接：`README.md`

## 下一次迭代建议

1. 先补真实二维码，再补更多景点，不要反过来。
2. 地点库建议按“高传播景点 + 高情绪诗句”优先，不要一上来铺太全。
3. 每次新功能都问一句：它是不是直接服务于“看起来更贵”。
4. 如果不是，就先不要加。

## 成功标志

- 用户第一次打开，不需要教程就知道怎么用
- 第一张图导出后，已经具备明显的“画册感”和“器材感”
- 页面全程没有登录、上传、等待接口的感觉

## 常见风险

- 原图没有 EXIF，信息感会下降
- 不同浏览器字体略有差异，排版会有微小变化
- 超大图片可能让低配手机导出变慢
